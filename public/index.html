Meine Abschlussarbeit an der CSAW Desk Booking System

Worum geht es ?

  • ist eine Single Page Application die das Buchen von Arbeitsplätzen ermöglicht
  • Flex Desk Buchung => maximal 3 Tage in Folge
  • Fix Desk Buchung => unbefristet
  • insgesamt 4 Räumlichkeiten mit 96 Arbeitsplätzen

Verwendete Technologien und Konzepte

  • Gitlab => gemeinsame Arbeit am Code und Projektplanung
  • Figma => Mockup und Designentwurf
  • React => JS Framework
  • Vanilla JavaScript => Logik
  • SASS => Styling
  • Postman => API Testen
  • Fetch API => Kommunikation mit dem Backend
  • React Loader Spinner => Ladevorgang visualisieren

Aufbau und Verwendung der App

Allgemeine Startseite

  • Registrierung
  • Anmeldung
  • Token und ID werden in das Local Storage gespeichert
  • Userdaten werden von der API geholt und auch Lokal gespeichert
  • Rollenbezogene Pfadweiterleitung
  • Solange Userdaten im Storage vorhanden sind findet die Weiterleitung bei jedem Aufruf der App statt

User Landing Page

  • Überblick über seine nächsten drei anstehenden Reservierungen
  • Navigation auf normalen User beschränkt

Reservierungen

  • Auflistung aller Reservierunge => getrennt in zukünftige und vergangene
  • Flex Desk Buchungen können bis 1 Tag vor der Reservierung storniert werden
  • bei vergangenen Buchungen können Kommentare abgegeben werden
  • bereits besuchte Tische kann man als Favorit markieren

Favoriten

  • Übersicht über alle Favoriten
  • Information ob Favorit gerade als Fix Desk in Verwendung ist => steht dann für eine Buchung nicht zur Verfügung
  • Möglichkeit Favorit aus Favoriten Liste zu löschen
  • Suchfunktion => schnelles Suchen nach freien Favoriten zu einen bestimmten Zeitpunkt

Tisch Buchen

  • Allgemeine Suche nach einen Arbeitsplatz
  • Verfügbare Tische enthalten Metainformationen
  • Raumübersicht => gibt Aufschluss wo sich der Arbeitsplatz im Raum befindet
  • Equipment => liefert Information über die Ausstatung des Tisches
  • Kommentare => Meinungen und Informationen anderer User über diesen Arbeitsplatz

Profil

  • Update seiner persönlichen Informationen
  • von der App abmelden

Admin Landing Page / Hot Zone

  • Schnellüberblick über unerledigte Aufgaben und Neuigkeiten
  • => neue Kommentare
  • => unbearbeitete Fix Desk Anfragen
  • => neue User
  • Admin Bereiche in der Navigation verfügbar

Kommentare

  • Übersicht sortiert nach alten und neuen Einträgen
  • aktuelle Kommentar Anzahl wird in das Local Storage gespeichert um später zwischen neuen und alten Kommentaren unterscheiden zu können

Fix Desk Requests

  • Liste aller unbeantworteten Fix Desk Anfragen inkl. Meta Informationen
  • => Raum und Arbeitsplatz
  • => Name des Users
  • => Datum der Anfrage
  • => Admin kann eine Anfrage genehmigen oder ablehnen

Userliste

  • Auflistung aller registrierter User sortiert nach Neu hinzugekommenen und älteren
  • Hinweis auf die Rolle eines Benutzers => Admin oder User
  • aktuelle User Anzahl wird in das Storage gespeichert um später zwischen neuen und alten Usern unterscheiden zu können

Im Fokus - die Routen der App

  • React Router
  • insgesamt 3 große Pfade
  • => nicht eingeloggte User
  • => normale User
  • => Admins
  • weitere Unter Routen für Administratoren und für User

Im Fokus - Booking Page

  • mit React States werden Programmabläufe gesteuert
  • es gibt zwei unterschiedliche Programmzweige im Code
    => einen für Fix Desk und einen für Flex Desk Buchungen
  • um verfügbare Tische zu finden wird
    => mit dem JS Date Objekt gearbeitet => den Array Methoden .filter() und .some() => mit der JS fetch API
  • Liste aller Arbeitsplätze vs augeschiedene Buchungen
